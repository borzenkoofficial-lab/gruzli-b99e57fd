

## План: Доработать дизайн ленты заявок

Сделаю ленту более «премиальной» и информативной, не ломая текущую структуру и свайп-логику. Все изменения в `src/screens/FeedScreen.tsx`.

### 1. Хедер — живее и информативнее
- Добавлю мини‑строку статуса под заголовком: пульсирующая зелёная точка + «N активных сейчас» (живой счётчик из `jobs.length`).
- Логотип Gruzli обёрну в мягкое свечение (тонкое кольцо + ring) для премиального акцента.
- Подсказку про свайпы превращу в маленький «чип» с иконками `←` `→`, не текстом.

### 2. Stats bar — стеклянные карточки с иконками
- Две карточки получат:
  - Цветную иконку слева в маленьком закруглённом контейнере (MapPin для «рядом», Wallet для «заработать»).
  - Микро‑sparkline/полоска снизу как декоративный акцент (тонкая линия с градиентом).
  - Лёгкий gradient‑border (через `bg-gradient-to-br` + inner card) для эффекта премиума.
- Анимация чисел при загрузке (плавное появление, без сторонних либ).

### 3. Фильтры — pill‑чипы с иконками и счётчиками
- Каждому фильтру добавлю иконку: «Все» (Sparkles), «Срочные» (Zap), «Быстрая минималка» (Clock).
- Активный фильтр получит лёгкое свечение / shadow и счётчик в скобках: `Срочные · 4`.
- Добавлю плавный layout‑переход (Framer `layout`).

### 4. Карточка заявки (`SwipeableJobCard`) — главное обновление
Сохраняю структуру (теги → заголовок → диспетчер → описание → pay block → meta → footer), добавляю детали:

- **Top‑accent bar**: тонкая 3px полоска сверху карточки — красная для `urgent`, зелёная для `quick_minimum`, нейтральная по умолчанию. Создаёт визуальную иерархию.
- **Бейджи (теги)**:
  - «Срочно» — мягкий градиент fill + лёгкое pulse-свечение для срочных.
  - «Быстрая минималка» — иконка Clock вместо просто текста.
  - Новый бейдж «Новое» (зелёная точка) для заявок созданных < 10 минут назад.
- **Диспетчер**: маленький круглый аватар-плейсхолдер (инициалы из `dispatcherName`) + имя + микро‑звезда рейтинга (если есть в profiles, иначе скрыть).
- **Pay block**: 
  - Лёгкий градиент `from-surface-1 to-surface-2`, тонкий accent‑border слева (1px зелёный).
  - Сумма крупнее (text-xl), с акцентом «₽» меньше и приглушённее.
  - Подпись «Ты получишь» → с мини‑иконкой TrendingUp.
- **Meta info**: каждый pill (адрес, метро, время, чел.) станет отдельным маленьким chip’ом с border вместо просто иконка+текст. Лучше скан‑чтение.
- **Footer**: 
  - Кнопка «Беру!» с градиентом `from-foreground to-foreground/80`, иконкой ArrowRight внутри, и hover‑shimmer (CSS).
  - При `responded` — компактный success‑state с галочкой в кружке.
- **Hover/Press**: карточка получает лёгкий `hover:border-foreground/20` и микро‑lift (translateY -1px).

### 5. Empty / Loading состояния
- Loading: 3 skeleton‑карточки вместо текста (используя существующий `ScreenSkeleton`‑паттерн).
- Empty: иллюстративный блок — большая иконка Search в круге, заголовок, подсказка + кнопка «Сбросить фильтры».

### 6. Микро‑детали
- Smart Search input получит подложку с тонким gradient ring при фокусе.
- Бейдж результатов поиска `Найдено: N` оформлю как pill.
- «Место занято» (бот) overlay сделаю красивее: затемнение карточки + центральная плашка вместо угла.

### Что НЕ меняю
- Логику свайпов, респонда, фильтрации, поиска, realtime‑подписки.
- Цветовую схему (использую только существующие токены: `surface-*`, `foreground`, `muted-foreground`, `online`, `destructive`, `border`).
- Структуру данных и пропсы.
